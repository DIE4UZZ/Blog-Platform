# CONSENSUS_login-register

## 明确需求描述
- 初始化全新前端工程，技术栈为 Vue3 + Tailwind CSS + Element Plus + Axios。
- 提供可独立运行的项目结构、路由配置与构建系统。
- 登录/注册页面遵循 Apple 官网风格的现代极简设计，响应式适配。
- 注册仅允许邮箱 + 密码，不包含用户名/手机号字段。
- 封装完整 API 访问层：请求/响应拦截、错误提示、加载状态管理。
- 忽略 Mac 环境要求，使用 Windows 环境配置与实现。

## 验收标准
- 项目可通过 `npm install` 与 `npm run dev` 启动并渲染登录/注册页面。
- 路由至少包含：`/login`、`/register`（可选 `/` 重定向）。
- 注册表单字段仅包含邮箱与密码，具备基础校验与错误提示。
- 登录表单包含邮箱/密码，具备校验与错误提示。
- API 模块具备：baseURL、请求/响应拦截、错误处理、统一返回结构解析。
- 页面在桌面与移动端显示正常，布局自适应。

## 技术实现方案与约束
- 构建工具：Vite。
- UI：Tailwind CSS + Element Plus（表单、按钮、提示）。
- 状态：页面级 loading 状态与提交禁用。
- API：Axios 封装，统一 error 处理与 token 管理预留。
- 代码规范：所有函数包含函数级注释（功能、参数、返回值、用途）。

## 任务边界限制
- 仅实现登录/注册页面，不实现其它业务页面。
- 不实现真实后端，仅对接 API 文档定义的接口。

## 未决项与处理
- API 注册接口文档包含 username/phone，但已按需求限制为邮箱注册；如后端强校验其他字段需调整。
- 视觉与字体可用近似方案实现（如选择与 Apple 风格相近的字体）。
